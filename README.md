# 🌐 VPN Gateway: 高可用家庭科学上网方案

## 1. 核心目标
建立一个基于 **阿派 (Local Gateway) + 老谷 (Google VM) + Cloudflare (CDN)** 的三位一体科学上网链路。
*   **低成本**：利用 Google Cloud 免费层及 Cloudflare 免费 CDN。
*   **高隐蔽**：隐藏老谷真实 IP，通过 WS+TLS 协议伪装流量。
*   **省钱策略**：利用 Cloudflare 规避 Google Cloud 昂贵的“直接出站至中国大陆”流量费。
*   **全家无感**：阿派作为热点/网关，实现家庭设备免客户端接入。

## 2. 流量拓扑与原理
### 2.1 流量路径
`[终端设备]` --(WiFi/热点)--> `[🐾 阿派 (192.168.0.105)]` --(加密隧道 WS+TLS)--> `[🟠 Cloudflare (CDN)]` --(转发)--> `[👴 老谷 (GCP VM)]` --(请求)--> `[🌍 目标互联网]`

### 2.2 核心组件角色
1.  **阿派 (本地枢纽)**：运行 v2rayA，负责流量截获、国内外分流、数据加密封装（WS+TLS）。
2.  **Cloudflare (流量护盾/中转)**：
    *   **隐藏 IP**：作为前置代理，阻断国内与老谷 IP 的直连，防止封锁。
    *   **流量节费**：利用 CF 的内网对等，将 Google 昂贵的“Google -> 中国大陆”直连流量，转化为“Google -> CF (同区/美国) -> 中国大陆”路径。
3.  **老谷 (公网出口)**：作为最终跳板，执行真实的互联网访问请求。

## 3. 部署路线图 (Roadmap)
- [ ] **阶段 1：老谷服务端部署** (Docker 部署 X-ui 或 v2ray，开启 WS 监听端口)。
- [ ] **阶段 2：域名与 CF 配置** (解析域名至老谷，开启“小云朵”Proxy 模式)。
- [ ] **阶段 3：阿派客户端对接** (配置 v2rayA 导入 CF 节点信息)。
- [ ] **阶段 4：热点与分流优化** (开启透明代理，优化 GeoIP/GeoSite 国内流量分流)。

## 4. 维护与监控
*   监控 `system.log` 查看连接延迟。
*   定期检查 Google Billing 账单，确认流量路径是否通过 CF 成功降低成本。
